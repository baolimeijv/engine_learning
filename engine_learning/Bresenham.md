# Bresenham (笔记)

## 1. 为什么代码里的 `err = dx - dy` 和学的 `p = 2*dy - dx` 不一样？
- 两者等价，只是变量代换。`err = dx - dy` 的写法能统一处理所有方向，避免分象限。

## 2. `e2 = 2 * err` 有什么用？
- 把误差项乘以2，将 `err > -dy/2` 的条件转为整数比较 `2*err > -dy`，保持纯整数运算。

## 3. 为什么同时有两个 `if` 判断？
- 两个 `if` 分别决定 x 和 y 方向是否移动，可以同时成立（对角线移动），从而自动适应任意斜率。

## 4. 初始 `err = dx - dy` 是怎么来的？
- 从标准决策参数 `p = 2*dy - dx` 变换而来，使得算法一开始就处于“中点”状态。

## 5. 这个代码能画所有方向的直线吗？
- 能。`sx`、`sy` 处理方向，两个 `if` 处理斜率，是 Bresenham 的通用实现。

## 6. 我需要背这个代码吗？
- 不需要。理解思想（整数误差、逐像素决策）即可，实际使用时直接复制。

## 7. 面试时问到怎么办？
- 能说出“用整数误差项决定下一个像素位置，避免浮点”就算过关，不必默写代码。

## 8. 在引擎开发中什么时候会再用到？
- 几乎不用。现代图形 API 和引擎已封装画线，它只是理解光栅化的练习。
