# shared_ptr 笔记（3.23）

## 1. 为什么引用计数要用 `size_t*` 放在堆上？
- 多个 `sharedptr` 对象必须共享同一个计数器，才能知道还有多少指针指向同一资源。
- 放在堆上，所有共享者都指向同一块内存，计数变化全局可见。

## 2. `operator*` 和 `operator->` 的作用是什么？
- `operator*` 返回所管理对象的引用，让 `*sp` 可以直接操作对象。
- `operator->` 返回原始指针，让 `sp->func()` 能调用对象的成员函数。

## 3. 拷贝构造和赋值分别做了什么？
- 拷贝构造：复制指针和计数器，并将引用计数加 1。
- 赋值：先释放当前资源，再复制新资源，并增加新资源的引用计数。

## 4. 为什么赋值前要检查自赋值？
- 如果 `a = a`，不加检查就会先 `release()` 释放资源，然后再尝试从自己复制，导致空指针访问。
- 正确写法：`if (this == &other) return *this;`

## 5. `release()` 中为什么要先判断 `count` 再解引用？
- 当对象管理空指针时，`count` 为 `nullptr`，直接 `*count` 会崩溃。
- 必须先 `if (count)`，再对 `*count` 操作。

## 6. 析构函数的作用是什么？
- 调用 `release()` 减少引用计数，当计数归零时释放管理的对象和计数器。

## 7. 为什么赋值运算符要返回 `*this`？
- 返回当前对象的引用，支持连续赋值（如 `a = b = c`），与内置类型行为一致。

## 8. 为什么 `count_t()` 要加 `const`？
- `count_t()` 只读，不修改对象，加 `const` 后可以在 `const` 对象上调用，语义更清晰。
